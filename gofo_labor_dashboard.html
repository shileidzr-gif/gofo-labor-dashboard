▶ 📖 概念与术语说明（点击展开）
术语含义
A类 · 核心稳定员工出勤≥8周、平均周工时≥20h、工时变异系数CV<0.4。现场最稳定可靠的骨干力量，出勤连续、工时饱满、表现稳定。
B类 · 高潜力员工出勤≥4周、平均周工时≥20h、工时变异系数CV<0.5。具备发展为A类核心员工的潜力，是重点培养对象。
C类 · 普通可用员工出勤≥3周、平均周工时≥10h。有一定出勤基础，能满足基本用工需求，可作为弹性补充劳动力。
D类 · 不稳定员工出勤2周及以上但不满足A/B/C条件。工时低或波动大，出勤不稳定，需要持续观察。
E类 · 低效出勤员工仅出勤1周。占用了招募、对接、培训资源后即消失，是劳务管理中的主要效率损失来源。
五维评分模型员工综合得分 = 出勤周数×30% + 平均周工时×25% + 出勤覆盖率×25% + 工时稳定性×20%。详见「评分模型说明」页面。
出勤覆盖率员工实际出勤周数 ÷ 总可用周数。衡量员工在整个分析周期中的出勤密度。
工时变异系数 (CV)员工各周工时的标准差 ÷ 平均周工时。CV越小越稳定，CV越大表示出勤波动剧烈。
核心率A类核心员工占供应商总派遣员工的比例。核心率越高，说明供应商输出稳定劳动力的能力越强。
高潜率B类高潜力员工占供应商总派遣员工的比例。高潜率反映供应商储备中层骨干人才的能力。
低质效率E类低效出勤员工占供应商总派遣员工的比例。低质效率越低越好，超过50%即亮红灯。
供应商综合评分满分100分 = 核心率×40 + 高潜率×20 + (1-低质效率)×20 + 工时水平/最高工时×20。详见「供应商质量分析」页面。
平均周工时员工在出勤的各周中，每周平均工作小时数。反映员工的劳动投入强度。

📌 关键发现
🚨 问题一：低质效率?E类员工占全部员工的比例。68.2%即10人中有近7人仅出勤1次高达68.2%
2,135名员工仅出勤1次即消失，占总员工数的68.2%。这些员工平均工时极低，却占用了大量招募、对接、现场培训资源，形成严重的资源浪费。

每个新员工的培训成本估算为0.5-2小时管理人员时间，2,135人低效出勤员工即消耗约1,000-4,000小时管理资源。
→ 建议动作：要求各供应商提交稳定员工名单，对低质效率超50%的供应商启动约谈整改机制。
✅ 优势：STARTPEOPLE贡献全部核心员工
8名A类核心稳定员工全部来自STARTPEOPLE，其B类高潜力员工也最多（44人）。该供应商虽然绝对低质效数量较多，但在核心员工沉淀能力上远超其他供应商，体现了更强的员工留存管理能力。
→ 建议动作：增加STARTPEOPLE的核心岗位派遣份额，探索与其建立核心员工池合作机制。
⚠️ 问题二：出勤人数高峰后断崖下跌
3月30日-4月5日周出勤人数达到峰值1,119人，但4月27日-5月3日仅剩471人，下降幅度58%。这种旺季大量补人、淡季大量流失的模式，导致稳定员工池无法建立，每次旺季都要重新大规模招募。
→ 建议动作：在淡季保留核心员工池200-300人，旺季优先调用，减少新员工比例。
💡 机会：103名A+B类员工是提升效率的杠杆?A+B类合计103人(8+95)，称为"核心+高潜"员工池，贡献约27%总工时
8名A类+95名B类员工合计103人，他们平均出勤7-10周，平均周工时超过20小时，工时稳定性高。优先保留和发展这个群体，可以显著提升现场效率，减少新员工培训压力。
→ 建议动作：将103人纳入"核心劳务工名单"，给予优先排班权、固定岗位、稳定出勤奖励。

⭐ A类 · 核心稳定员工 (8人 · 0.3%)
连续出勤8-10周，周均工时≥20h，工时高度稳定（CV<0.4）。全部来自STARTPEOPLE，熟悉现场流程，是仓库运营的中坚力量。本期代表：THIAM Sougalo (258h,9周)、BAYIKA Cyrille (246h,9周)。
→ 管理动作：纳入核心员工池，优先排班，禁止供应商随意替换，设立稳定出勤专项奖励。
🚀 B类 · 高潜力员工 (95人 · 3.0%)
出勤周数4-10周，周均工时≥20h，稳定性较好。部分员工工时总量甚至超过A类，仅因出勤时长较短未达A级门槛。如KONATE Samba (282h,10周)是典型代表，应重点追踪转化。
→ 管理动作：第2、4、8周设置留存确认节点，主动沟通需求，固定岗位和班次。
🟡 C类 · 普通可用员工 (490人 · 15.7%)
出勤3周以上，周均工时10-20h，能满足基本出勤需求，但工时或稳定性一般。可作为旺季的补充力量，部分员工有转化为B类的潜力。
→ 管理动作：维持当前合作关系，对工时较高者给予关注，提供改善激励。
⚡ D类 · 不稳定员工 (403人 · 12.9%)
出勤2周但工时低或波动大，不适合安排关键岗位。可能是工作强度不符合期望、交通不便或主观意愿不强导致的短期出勤现象。
→ 管理动作：排查离场原因，调整岗位/班次后观察，无改善则不再优先安排。
❌ E类 · 低效出勤员工 (2,135人 · 68.2%)
仅出勤1次，对现场贡献极低，却消耗了巨量管理和培训资源。这类员工的高占比是当前最大的管理痛点，直接原因可能包括：供应商无效招募、岗位与员工预期不符、现场体验差等。
→ 管理动作：要求供应商对低质效率超50%进行书面说明，引入低质效率KPI考核，与订单份额挂钩。

⚡ 关键趋势解读
📈 快速增长期（3月2日—4月5日）
从3月2日的310人快速攀升至4月5日的1,119人，5周内增长261%。这一增长主要由STAFFMATCH贡献（从135人增至624人），反映了旺季临时大规模补人的模式，但STAFFMATCH新增员工中低质效占比高，工时贡献效率偏低。
📉 断崖式下滑（4月13日—5月3日）
从4月13日峰值885人降至5月3日的471人，3周内下降47%。这种断崖式下跌通常意味着项目进入尾期，大量临时工离场。问题在于：核心员工是否也随之流失？是否建立了淡季留存机制？
⚠️ MANPOWER入场时机异常
MANPOWER在3月30日旺季高峰才首次大规模出现（70人），此前几乎不存在。作为新引入供应商，低质效率高达78.8%，说明临时紧急招募质量较差。建议提前规划供应商入场时间，而非旺季突击补人。
✅ STARTPEOPLE稳定性最佳
STARTPEOPLE从第1周至第10周全程出勤，无断层，并贡献了全部A类核心员工。其周度人数增减较平稳，反映了更有计划的人员调度能力，是当前最值得信赖的供应商。

📐 员工评分模型完整公式
总评分 = score_weeks×30% + score_hours×25% + score_attendance×25% + score_stability×20% 其中： score_weeks = (员工出勤周数 / 数据期内最大出勤周数) × 100 score_hours = (员工平均周工时 / 数据期内最高平均周工时) × 100 score_attendance = 员工出勤周数 / 总周次数 × 100 score_stability = (1 - CV/2) × 100，CV = 工时标准差 / 平均工时
员工分类规则（优先级从高到低）：
• A类（核心稳定）：出勤周数≥8 且 周均工时≥20h 且 CV<0.4
• B类（高潜力）：出勤周数≥4 且 周均工时≥20h 且 CV<0.5
• C类（普通可用）：出勤周数≥3 且 周均工时≥10h
• D类（不稳定）：出勤周数=2 且 (周均工时<10h 或 CV≥0.5)
• E类（低质效）：出勤周数=1
🏢 供应商评分模型完整公式
供应商评分 = core_rate×40 + potential_rate×20 + (1-trash_rate)×20 + (avg_weekly_hours/max_hours)×20 其中： core_rate = A类员工数 / 供应商总员工数 potential_rate = B类员工数 / 供应商总员工数 trash_rate = E类员工数 / 供应商总员工数 avg_weekly_hours = 供应商所有周次的平均工时
供应商分类规则：
• A类（战略优质）：评分≥35 且 低质效率<40%
• B类（稳定可用）：评分≥25 且 低质效率<60%
• C类（补充型）：评分≥15
• D类（风险）：评分≥8
• E类（低质效）：评分<8
